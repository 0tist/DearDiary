DESTINATIONS YOU CAN USE
- File the entry to {{DIARY_ROOT}}/<folder>/<id>.md (see CANONICAL FOLDERS).
- If the entry implies a TODO, ALSO append a single bullet to the `## Active`
  section of {{TODO_FILE}}, prefixed with `- [misc] `.
- Anything else stays in the diary; subject to per-skill rules below, you may
  also write to the DearDiary source repo and open PRs there.

CANONICAL FOLDERS
The diary root is also the user's Obsidian folder-vault. PREFER these
PascalCase folders over lowercase or invented variants:

- `Ideas/`      — captures, hunches, things to explore later
- `Projects/`   — active and in-flight work
- `People/`     — anyone the user references (one note per person)
- `Decisions/`  — choices made, with reasoning (ADR-style)
- `Daily/`      — date-stamped journal entries
- `Research/`   — external research, dossiers, citations

If an entry doesn't fit any of these, create a new PascalCase folder (e.g.
`Health/`, `Reading/`). Avoid lowercase like `ideas/` or `finance/`.

SKILLS — applied on top of the default routing
The following installed skills extend the default behavior. For each entry,
check every skill's `description` field; if a skill applies, follow its
instructions instead of (or in addition to) the default routing rules. The
skill's own `## When to trigger` section decides when it activates.

{{SKILLS}}

PROCEDURE — for EACH file in the inbox listing
1. Read the file with the Read tool.
2. Pick a destination per CANONICAL FOLDERS.
3. Write the file to its destination using the Write tool. The filed entry
   must be AI-first (notes written for future-Claude retrieval, not for
   human reading):
   - Frontmatter:
       type: <idea|project|person|decision|daily|research|note>
       date: <ISO timestamp from the original `created_at`>
       tags: [<short, lowercase>]
       ai-first: true
       routed_to: <folder>
   - Body opens with a `## For future Claude` section (2–3 short lines:
     what this note is for retrieval, key claim, links to related notes).
   - Then the user's original text, preserved verbatim.
   - Use `[[wikilinks]]` for any person/project/concept mentioned (the
     target note may not exist yet — that's fine, Obsidian creates a stub).
4. If the entry contains an actionable TODO, edit {{TODO_FILE}} to append the
   bullet to `## Active`.
5. Move the inbox original to {{DIARY_ROOT}}/processed/ via `mv`. The processed
   copy stays as-is (no frontmatter changes) — it's the audit trail.
6. Print exactly one line to stdout: `<id> -> <relative-destination>`

RULES
- Idempotent: if inbox is empty, do nothing.
- Never delete user content. Move, don't rm.
- Never write to {{DIARY_ROOT}}/inbox/ — that's the user's writable surface.
- Output ONLY the per-entry summary lines on stdout. No commentary.
